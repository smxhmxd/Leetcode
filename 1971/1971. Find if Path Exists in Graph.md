# ğŸ¯Problem Explaination:

TheÂ `Find if Path Exists in Graph`Â problem involves determining whether there is a path between two given nodes (`source`Â andÂ `destination`) in an undirected graph. The graph is represented byÂ `n`Â nodes and a list ofÂ `edges`, where each edge specifies a connection between two nodes.

**Given:**

- `n`: Number of nodes in the graph.
- `edges`: List of edges where each edge is represented asÂ `[u, v]`, indicating an undirected edge between nodeÂ `u`Â and nodeÂ `v`.
- `source`: The starting node for the path.
- `destination`: The target node to reach.

**Objective:**  
You need to implement a solution that can determine if there exists a path from theÂ `source`Â node to theÂ `destination`Â node in the given graph.

**Output:**  
ReturnÂ `True`Â if there exists a path fromÂ `source`Â toÂ `destination`, otherwise returnÂ `False`.

# 1ï¸âƒ£Depth-First Search (DFS)ğŸ¤“:

1. **Graph Representation:**
    
    - Construct an adjacency list representation of the graph using the givenÂ `edges`. This representation allows efficient traversal of nodes and their neighbors.
2. **Depth-First Search (DFS) Function:**
    
    - Implement a recursive DFS function (`dfs`) to traverse the graph starting from theÂ `source`Â node.
    - Use aÂ `visited`Â set to keep track of nodes that have been visited during the traversal to avoid revisiting and potential infinite loops (cycles).
    - Base case: If the current node being visited is theÂ `destination`, returnÂ `True`Â as a path has been found.
    - Recursive case: For each neighbor of the current node, if it has not been visited yet, recursively callÂ `dfs`Â on that neighbor.
    - If any recursive call returnsÂ `True`, propagateÂ `True`Â back up through the call stack indicating that a path to theÂ `destination`Â has been found.
    - If the DFS traversal completes without finding a path to theÂ `destination`, returnÂ `False`.
3. **Execution:**
    
    - Invoke theÂ `dfs`Â function starting from theÂ `source`Â node and return the result.

```cpp
class Solution {
public:
    bool validPath(int n, vector<vector<int>>& edges, int source, int destination) {
        unordered_map<int, vector<int>> graph;
        for (const auto& edge : edges) {
            int u = edge[0];
            int v = edge[1];
            graph[u].push_back(v);
            graph[v].push_back(u);
        }
        
        unordered_set<int> visited;
        return dfs(source, destination, graph, visited);
    }
    
    bool dfs(int node, int destination, unordered_map<int, vector<int>>& graph, unordered_set<int>& visited) {
        if (node == destination) {
            return true;
        }
        visited.insert(node);
        for (int neighbor : graph[node]) {
            if (visited.find(neighbor) == visited.end()) {
                if (dfs(neighbor, destination, graph, visited)) {
                    return true;
                }
            }
        }
        return false;
    }
};
```

# 2ï¸âƒ£Breadth-First Search (BFS)ğŸ˜:

1. **Graph Representation:**
    
    - Similarly, construct an adjacency list representation of the graph using the givenÂ `edges`.
2. **Breadth-First Search (BFS) Function:**
    
    - Utilize a queue (`deque`) to perform a level-order traversal starting from theÂ `source`Â node.
    - Use aÂ `visited`Â set to keep track of nodes that have been visited during the traversal.
    - Begin BFS by enqueueing theÂ `source`Â node into the queue and marking it as visited.
    - While the queue is not empty, dequeue a node, check if it is theÂ `destination`, and returnÂ `True`Â if so.
    - Otherwise, enqueue all unvisited neighbors of the dequeued node into the queue and mark them as visited.
    - If the BFS traversal completes without finding theÂ `destination`, returnÂ `False`.
3. **Execution:**
    
    - Invoke the BFS function starting from theÂ `source`Â node and return the result.

```cpp
class Solution {
public:
    bool validPath(int n, vector<vector<int>>& edges, int source, int destination) {
        unordered_map<int, vector<int>> graph;
        
        for (auto& edge : edges) {
            int u = edge[0];
            int v = edge[1];
            graph[u].push_back(v);
            graph[v].push_back(u);
        }
        
        queue<int> queue;
        unordered_set<int> visited;
        
        queue.push(source);
        visited.insert(source);
        
        while (!queue.empty()) {
            int node = queue.front();
            queue.pop();
            if (node == destination) {
                return true;
            }
            for (int neighbor : graph[node]) {
                if (visited.find(neighbor) == visited.end()) {
                    visited.insert(neighbor);
                    queue.push(neighbor);
                }
            }
        }
        
        return false;
    }
};
```

# 3ï¸âƒ£Union-Find (Disjoint Set Union)ğŸ˜:

1. **Initialization:**
    
    - Initialize aÂ `parent`Â array where each node initially points to itself (each node is its own parent).
    - Use aÂ `rank`Â array to keep track of the rank (or depth) of each node's subtree for efficient union operations.
2. **Union-Find Operations:**
    
    - ImplementÂ `find`Â andÂ `union`Â operations to support efficient checking and merging of disjoint sets (connected components) in the graph.
    - TheÂ `find`Â operation utilizes path compression to find the root (or representative) of the set containing a given node.
    - TheÂ `union`Â operation merges two sets by linking their roots based on theÂ `rank`Â to keep the tree balanced.
3. **Connect Components:**
    
    - Iterate through each edgeÂ `(u, v)`Â in theÂ `edges`Â list and perform theÂ `union`Â operation to connect nodesÂ `u`Â andÂ `v`.
4. **Path Existence Check:**
    
    - After connecting all nodes based on the edges, use theÂ `find`Â operation to check if theÂ `source`Â andÂ `destination`Â nodes belong to the same connected component (set).
    - If they do, a path exists betweenÂ `source`Â andÂ `destination`; otherwise, it does not.

```python
class Solution:
    def validPath(self, n: int, edges: List[List[int]], source: int, destination: int) -> bool:
        parent = list(range(n))
        rank = [1] * n
        
        def find(x):
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]
        
        def union(x, y):
            rootX = find(x)
            rootY = find(y)
            if rootX != rootY:
                if rank[rootX] > rank[rootY]:
                    parent[rootY] = rootX
                elif rank[rootX] < rank[rootY]:
                    parent[rootX] = rootY
                else:
                    parent[rootY] = rootX
                    rank[rootX] += 1
        
        for u, v in edges:
            union(u, v)
        
        return find(source) == find(destination)
```

# 4ï¸âƒ£Using Standard Graph Algorithms

1. **Graph Representation:**

2. **Dijkstra's Algorithm (or Similar):**
    
    - Implement a shortest path algorithm (e.g., Dijkstra's algorithm) to find the shortest paths from theÂ `source`Â node to all other nodes in the graph.
    - Initialize aÂ `distances`Â dictionary to keep track of the shortest known distance from theÂ `source`Â to each node.
    - Use a priority queue (min-heap) to efficiently explore nodes in order of their current shortest distance from theÂ `source`.
    - Begin with theÂ `source`Â node having a distance ofÂ `0`Â in the priority queue.
    - Continuously dequeue nodes from the priority queue, update distances to neighboring nodes, and enqueue them if a shorter path is found.
    - If theÂ `destination`Â node is dequeued from the priority queue, a path fromÂ `source`Â toÂ `destination`Â has been found, and returnÂ `True`.
3. **Path Existence Check:**
    
    - After the algorithm completes, check if theÂ `destination`Â node was reached during the execution.
    - If so, a path exists fromÂ `source`Â toÂ `destination`; otherwise, it does not.

```cpp
class Solution {
public:
    bool validPath(int n, vector<vector<int>>& edges, int source, int destination) {
        unordered_map<int, vector<int>> graph;
        for (const auto& edge : edges) {
            int u = edge[0];
            int v = edge[1];
            graph[u].push_back(v);
            graph[v].push_back(u);
        }
        
        vector<int> distances(n, numeric_limits<int>::max());
        distances[source] = 0;
        
        priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> priorityQueue;
        priorityQueue.push({0, source});
        
        while (!priorityQueue.empty()) {
            auto current = priorityQueue.top();
            priorityQueue.pop();
            int currentDistance = current.first;
            int currentNode = current.second;
            
            if (currentNode == destination) {
                return true;
            }
            
            if (currentDistance > distances[currentNode]) {
                continue;
            }
            
            for (int neighbor : graph[currentNode]) {
                int distance = currentDistance + 1; // Assuming unweighted graph
                if (distance < distances[neighbor]) {
                    distances[neighbor] = distance;
                    priorityQueue.push({distance, neighbor});
                }
            }
        }
        
        return false;
    }
};
```
